Android下获取当前XML的根View

发表于 2016-09-22 | 分类于 Android学习 |

首先可以看到getWindow().getDecorView()和getDecorView().getRootView()获得的View和id是一样的， 但是mBtn2.getRootView()获取的却是LinearLayout，而mBtn2控件刚好在根视图为LinearLayout的xml文件里面，而如果直接使用mBtn2 = (Button) findViewById(R.id.btn2)，然后再调用mBtn2.getRootView()，却可以得到与之前一致的结果，查询LayoutInflater的API文档后，第一行的描述是这样的：Instantiates a layout XML file into its corresponding View objects，也就说通过这个方法，它相当于单独创建了一个View的结构图，而这个根视图就是LinearLayout，而如果不使用这种方式所得到应该是当前Activity的根视图，所以我们如果想获得当前XML的根视图，参照上面的结构图我们需要先找到custome View，所以通过mDecorView.findViewById(android.R.id.content)获取到了用户自定义的View，但是查看输出结果可以看见类型是ContentFrameLayout而不是RelativeLayout，所以实际上在用户定义的xml的视图外层实际上应该还有一层视图包裹着，这个时候如果要获得XML的根View，只需转换成ViewGroup类型调用getChildAt(0)方法即可。

总结：

获取XML根View：
　　View rootView = ((ViewGroup) (getWindow().getDecorView().findViewById(android.R.id.content))).getChildAt(0);
getDecorView():

　　Retrieve the top-level window decor view (containing the standard window frame/decorations and the client’s content inside of that), which can be added as a window to the window manager.
getRootView():

　　Finds the topmost view in the current view hierarchy.
